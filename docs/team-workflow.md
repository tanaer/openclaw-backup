# 技术研发团队工作流程

## 📋 任务处理流程

### 阶段 1: 需求分析 (接到任务后首先执行)
1. **编写需求文档** - `docs/requirements/YYYY-MM-DD-task-name.md`
2. **明确验收标准** - 可测试的检查清单
3. **风险评估** - 识别潜在技术风险

### 阶段 2: 技术设计
1. **架构设计** - 组件图、数据流、接口定义
2. **任务拆分** - 拆分为可独立测试的子任务
3. **角色分配** - 使用 sub-agent 分配任务

### 阶段 3: 开发执行
```
执行者 (Executor): code-agent (GLM-5)
职责：实现功能、编写单元测试
```

### 阶段 4: 代码审查
```
审查者 (Reviewer): reviewer-agent (qwen3-max-2026-01-23)
职责：
  - 代码质量检查
  - 安全漏洞扫描
  - 性能问题识别
  - 不符合需求的问题
```

### 阶段 5: 测试验证
```
测试员 (Tester): test-agent
职责：
  - 功能测试 - 验证需求文档所有要求
  - 集成测试 - 验证模块间协作
  - 边界测试 - 极端情况测试
  - 回归测试 - 确保不影响现有功能
```

### 阶段 6: 安全检查
```
安全员 (Security): 专门检查
职责：
  - SQL 注入
  - XSS 攻击
  - CSRF 防护
  - 认证授权
  - 敏感数据保护
```

### 阶段 7: 交付确认
1. **客户验收** - 用户确认符合需求
2. **文档更新** - README、API 文档、部署指南
3. **备份归档** - GitHub 提交 + 标签
