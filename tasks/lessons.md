记录从用户纠正中学到的教训，避免重复犯错。

## 2026-02-24

### 1. 任务完成必须做3次彻底检查
**错误**: 完成任务后没有充分验证就报告完成
**纠正**: 每个任务完成后必须做3次定时检查，从不同角度彻底检查系统
**规则**: 
- 检查1: 功能是否正常工作
- 检查2: 边界情况和错误处理
- 检查3: 代码质量和最佳实践

### 2. 长任务必须定期汇报进度
**错误**: 执行长任务时没有汇报进度
**纠正**: 每 5-10 分钟汇报一次进展
**规则**: 使用 `⏱️ 进度 [XX%]` 格式定期汇报

### 3. 开源项目和开发项目要区分
**错误**: 混淆了开源项目和开发项目的部署
**纠正**: 
- 开源项目 (`hub.claudeai.best`): 直接部署 GitHub 官方代码
- 开发项目 (`dev.claudeai.best`): 自定义开发版本
**规则**: 两个项目用不同域名，不同端口

### 4. 语言包必须完整
**错误**: 中文界面显示英文，部分 key 缺失
**纠正**: 确保中英文语言包完整，所有 key 都有翻译
**规则**: 添加新功能时必须同时更新 `en.json` 和 `zh-CN.json`

### 5. 品牌名称统一
**错误**: 界面上还有 AxonHub 没改成 MuskAPI
**纠正**: 所有界面上的 AxonHub 必须改成 MuskAPI
**规则**: 检查所有 `.tsx`, `.ts`, `.json`, `.html` 文件

### 6. 按钮对比色问题
**错误**: `variant="outline"` 按钮在深色背景上看不清
**纠正**: 使用自定义样式 `bg-white/10 text-white border border-white/30 hover:bg-white/20`
**规则**: 在深色背景上使用半透明背景 + 白色文字 + 边框

### 7. Landing Page 登录状态
**错误**: Landing Page 没有检查登录状态，始终显示"登录"按钮
**纠正**: 使用 `useAuthStore` 检测 `accessToken`，已登录显示"控制台"按钮
**规则**: 所有公开页面都要检查登录状态，提供不同的导航选项

## 模式总结

### 前端开发
- 深色背景按钮: `bg-white/10 text-white border border-white/30 hover:bg-white/20`
- 登录状态检测: `const { accessToken } = useAuthStore(); const isLoggedIn = !!accessToken;`
- 语言包: 同时更新 `en.json` 和 `zh-CN.json`

### 部署
- 开源项目: 官方代码，独立端口
- 开发项目: 自定义代码，独立端口
- 域名区分: 不同子域名指向不同服务

### 工作流程
- 计划 → 实现 → 验证(3次) → 汇报
- 长任务: 每 5-10 分钟汇报进度
